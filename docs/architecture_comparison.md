# Dobot VLM/VLA 파이프라인 아키텍처 설계

## 전체 아키텍처

```
┌─────────────────────────────────────────────────────┐
│                    사용자 음성                        │
│              "빨간 컵 좀 집어줘"                      │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  1. STT — Qwen2.5-Omni-7B (ASR)                     │
│     음성 → 텍스트: "빨간 컵 좀 집어줘"                  │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  2. LLM — 태스크 정규화 + 판단                         │
│     "빨간 컵 좀 집어줘"                                │
│         → task: "pick up the red cup"                │
│         → params: {object: "red cup", action: "pick"}│
└──────────────────────┬───────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  3. VLA — Pi0                                        │
│     카메라(top+front) + state + task                  │
│         → delta action [dx, dy, dz, dr, grip]        │
└──────────────────────┬───────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────┐
│  4. Dobot Magician 실행                               │
└──────────────────────────────────────────────────────┘
```

## 모델 선정

| 역할 | 선정 모델 | 선정 이유 |
|------|----------|----------|
| STT | Qwen2.5-Omni-7B | 한국어 ASR 성능 우수, 최신 모델 |
| LLM | Qwen3-8B | 한국어 강점, 가벼움, Function Call/Thinking 모드 지원 |
| VLA | Pi0 (LeRobot) | PaliGemma 기반, LeRobot v3.0 호환, 단일 팔 로봇 지원 |

## LLM 비교: Gemma4 vs Qwen3

### 성능 비교

| 항목 | Gemma4 (12B) | Qwen3 (8B) | Qwen3 (32B) |
|------|-------------|------------|-------------|
| 한국어 | 보통 | **우수** | **우수** |
| 영어 | 우수 | 우수 | 우수 |
| 멀티모달 | **비전 지원** | 텍스트 전용 | 텍스트 전용 |
| 모델 크기 | 12B | **8B (가벼움)** | 32B (무거움) |
| Mac 48GB 구동 | 가능 | **쉬움** | 빡빡함 |
| Pi0와 궁합 | Gemma 생태계 통일 | 별도 생태계 | 별도 생태계 |
| 추론 속도 | 보통 | **빠름** | 느림 |
| Function Call | 지원 | **강력한 지원** | **강력한 지원** |
| Thinking 모드 | 없음 | **있음** | **있음** |

### Gemma4 장단점

**장점:**
- Pi0가 Gemma(PaliGemma) 기반이라 생태계가 통일됨
- 멀티모달(비전) 지원 — 이미지 입력 가능
- Google 생태계 연동 용이

**단점:**
- 한국어 성능이 Qwen3 대비 떨어짐
- 12B로 Qwen3-8B보다 무거움
- Thinking 모드 없음 — 복잡한 지시 분해에 불리

### Qwen3 장단점

**장점:**
- 한국어 성능 우수 (학습 데이터에 한국어 비중 높음)
- 8B 모델로 가벼워 Mac에서 Pi0와 동시 구동 가능
- Thinking 모드 — "빨간 컵 저기 왼쪽에 놓아줘" 같은 복잡한 지시를 단계별로 분해
- Function Call 강력 지원 — 태스크 정규화를 구조화된 출력으로 처리

**단점:**
- 텍스트 전용 (비전 필요 시 Qwen3-VL 별도 사용)
- Pi0(Gemma 기반)와 생태계가 다름

### 결론: Qwen3-8B 선정

이 파이프라인에서 LLM의 역할은 **텍스트→텍스트 변환**(태스크 정규화)이므로 비전이 필요 없음.
한국어 사용자를 대상으로 하므로 한국어 성능이 우선이고, Pi0와 동시 구동해야 하므로 8B가 적합.

## 메모리 배분 (Mac Apple Silicon 48GB)

```
Pi0 (3B, float32)         ~12GB
Qwen3-8B (Q4 양자화)        ~5GB
Qwen2.5-Omni-7B (Q4 양자화)  ~5GB
시스템 + 여유                ~26GB
──────────────────────────────────
합계                        ~22GB  (여유 있음)
```

- 세 모델 동시 로딩 가능 (양자화 필수)
- float16으로 올리면 메모리 부족 가능 → Q4/Q8 양자화 권장

## 각 모델의 역할 상세

### 1. STT (Qwen2.5-Omni-7B)
- 마이크 입력 → 한국어/영어 텍스트 변환
- 실시간 스트리밍 ASR 가능

### 2. LLM (Qwen3-8B)
- 자유로운 사용자 발화를 Pi0가 이해할 수 있는 표준 태스크 문자열로 변환
- 예시:
  - "저기 빨간 컵 좀 집어줘" → "pick up the red cup"
  - "컵 왼쪽에 놓아" → "place the cup on the left"
- 복잡한 지시를 여러 단계 태스크로 분해 가능 (Thinking 모드 활용)
- 등록되지 않은 태스크 요청 시 사용자에게 안내

### 3. VLA (Pi0)
- 카메라 이미지(top + front) + 로봇 상태 + 태스크 문자열 → 로봇 액션 출력
- delta action [dx, dy, dz, dr, grip] 형태
- LeRobot v3.0 데이터셋으로 파인튜닝

## VLA 모델 비교 (참고)

| 모델 | 베이스 VLM | 공개 여부 | LeRobot 지원 |
|------|-----------|----------|-------------|
| **Pi0** | PaliGemma (Gemma 2B) | 공개 | 지원 |
| OpenVLA | Prismatic (SigLIP + DinoV2) | 공개 | 일부 지원 |
| RT-2 | PaLM-E | 비공개 | 미지원 |

Pi0 선정 이유: LeRobot 프레임워크 공식 지원, 데이터 수집-학습-추론 파이프라인이 완성되어 있음.

## 카메라 구성

| 위치 | 카메라 | 역할 |
|------|--------|------|
| Top | 웹캠 또는 RealSense (RGB) | 작업 공간 전체 뷰 |
| Front | 웹캠 또는 RealSense (RGB) | 그리퍼-물체 근접 뷰 |

- Depth는 학습에 사용하지 않음 (Pi0는 RGB만 입력)
- RealSense 사용 시 RGB만 활용 (고해상도 이점)
