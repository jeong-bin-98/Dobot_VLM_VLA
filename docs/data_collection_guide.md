# Dobot Magician 데이터 수집 가이드

## 1. 준비물

| 항목 | 설명 |
|------|------|
| Dobot Magician | 전원 어댑터 + USB 케이블 연결 |
| 카메라 2대 | 웹캠 또는 RealSense (RGB만 사용) |
| 삼각대/거치대 | 카메라 고정용 (최소 2개) |
| 물체 | 학습에 사용할 대상 (컵, 블록 등) |
| 테이프 | 작업 영역 표시용 (권장) |

## 2. 환경 설정

```bash
# conda 환경 활성화
conda activate lerobot

# 의존성 설치 (최초 1회)
pip install -r requirements.txt
```

## 3. 카메라 배치

```
          [Top 카메라] ← 위에서 아래로 (40~60cm 높이)
               |
               v
      ┌────────────────┐
      │ ┌────────────┐ │
      │ │  작업 영역  │ │  ← 테이프로 표시 (약 20x20cm)
      │ └────────────┘ │
      └────────────────┘
            [Dobot]

      [Front 카메라] ← 정면에서 약간 비스듬히 (30~45도)
```

- **Top (cam1)**: 작업 공간 전체가 프레임에 들어오도록 위에서 촬영
- **Front (cam2)**: 그리퍼와 물체가 잘 보이는 정면/측면 각도
- 두 카메라 모두 **수집 중 절대 움직이지 않도록** 단단히 고정

## 4. 카메라 ID 확인

```bash
python -c "import cv2; [print(f'cam {i}: {cv2.VideoCapture(i).isOpened()}') for i in range(5)]"
```

`True`가 나오는 번호가 사용 가능한 카메라입니다.
어떤 번호가 어떤 카메라인지 하나씩 열어서 확인하세요.

## 5. 실행

```bash
python scripts/01_collect_data.py \
    --cam1 0 \
    --cam2 1 \
    --task "pick up the red cup" \
    --save_dir ./dataset_v3
```

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--cam1` | Top 카메라 ID | 0 |
| `--cam2` | Front 카메라 ID | 1 |
| `--task` | 태스크 설명 (영어) | pick_object |
| `--save_dir` | 저장 경로 | dataset_v3 |
| `--port` | Dobot 시리얼 포트 (보통 자동 감지) | 자동 |
| `--fps` | 기록 FPS | 5 |

## 6. 키 조작

실행하면 카메라 프리뷰 창이 뜹니다. **반드시 프리뷰 창을 클릭한 상태에서** 키를 눌러야 합니다.

| 키 | 동작 |
|----|------|
| **S** | 스텝 시작 (현재 카메라 이미지 + 로봇 위치 캡처) |
| **E** | 스텝 끝 (이동량 delta 기록) |
| **G** | 그리퍼(흡착컵) ON/OFF 토글 |
| **Z** | 손목 왼쪽 회전 |
| **C** | 손목 오른쪽 회전 |
| **V** | 현재 에피소드 저장 |
| **D** | 현재 에피소드 버리기 |
| **R** | 리플레이 (로봇이 기록된 동작 재현) |
| **H** | 홈 위치로 복귀 |
| **ESC** | 종료 |

## 7. 한 에피소드 수집 과정 (예: 컵 집기)

하나의 에피소드 = 하나의 태스크 완수 과정입니다.

```
1. [S] 누름         ← 현재 상태 캡처
2. 손으로 팔을 컵 위로 이동
3. [E] 누름         ← 이동량 기록

4. [S] 누름
5. 팔을 아래로 내려서 컵에 닿게
6. [E] 누름

7. [G] 누름         ← 그리퍼 ON (컵 잡기)

8. [S] 누름
9. 팔을 위로 들어올림
10. [E] 누름

11. [S] 누름
12. 목표 위치로 팔 이동
13. [E] 누름

14. [G] 누름        ← 그리퍼 OFF (컵 놓기)

15. [V] 누름        ← 에피소드 저장!
```

## 8. 반복

- 위 과정을 **같은 태스크로 50~100 에피소드** 반복합니다.
- 에피소드마다 물체 위치를 **약간씩 다르게** 배치하면 모델의 일반화 성능이 좋아집니다.
- 처음에는 좁은 영역에서 시작하고, 점차 범위를 넓혀가세요.

## 9. 수집 팁

- **일관된 조명**: 수집 중 조명이 바뀌지 않도록
- **천천히 움직이기**: 급하게 하면 delta 값이 튀어서 학습 품질이 떨어짐
- **실패한 에피소드는 [D]로 버리기**: 잘못된 데이터는 학습에 악영향
- **[R]로 리플레이 확인**: 저장 전에 로봇이 제대로 재현하는지 확인
- **물체 위치 변화**: 매 에피소드마다 시작 위치를 조금씩 바꿔서 다양성 확보

## 10. 저장 구조

수집이 완료되면 다음과 같은 LeRobot v3.0 형식으로 저장됩니다:

```
dataset_v3/
├── meta/
│   ├── info.json              # 데이터셋 메타정보
│   ├── stats.json             # 통계 (정규화용)
│   ├── tasks.jsonl            # 태스크 목록
│   └── episodes/
│       └── chunk-000/
│           └── episodes.parquet
├── data/
│   └── chunk-000/
│       ├── episode_000000.parquet
│       ├── episode_000001.parquet
│       └── ...
└── images/
    ├── observation.images.top/
    │   └── chunk-000/
    │       ├── episode_000000/
    │       │   ├── frame_000000.jpg
    │       │   └── ...
    │       └── ...
    └── observation.images.front/
        └── chunk-000/
            └── ...
```

## 11. 트러블슈팅

| 문제 | 해결 |
|------|------|
| Dobot 연결 실패 | 전원 어댑터 연결 확인. USB만으로는 응답 안 함 |
| 잘못된 포트 감지 | `--port /dev/cu.usbserial-XXXX` 직접 지정 |
| 카메라 안 열림 | `--cam1`, `--cam2` 번호 바꿔보기 |
| 키 입력 안 됨 | 카메라 프리뷰 창을 클릭한 후 입력 |
| 프리뷰 느림 | 정상 동작. 포즈 갱신은 5프레임마다 수행 |
